# Codex Change Log

Short log for code and architecture changes made by Codex.

AI change protocol:

```text
Before any AI-made code or experiment edit:
1. Read this changelog first.
2. State what will be added, deleted, or modified.
3. State whether the edit conflicts with the protected architecture below.
4. State the expected impact and risk.
5. After the edit, add a dated changelog entry describing the actual change.
```

Current protected architecture:

```text
stage1 frozen BERT latent autoencoder
stage2 Riemannian metric flow over latent space
FlowNet + MetricNet
natural velocity / metric-aware sampling
```

Framework interpretation:

```text
Option B: true natural-velocity parameterization.

FlowNet predicts a covector / force-like field f_theta.
MetricNet predicts a positive diagonal metric G=diag(g).
The latent ODE uses natural velocity v = G^{-1} f_theta, implemented as f_theta / g.
Training targets the matching covector G u_t, where u_t is the local Euclidean
residual direction toward the real suffix latent.
```

This means metric division at rollout and inference is intentional. Future flow
losses should supervise the force/covector field, not a raw Euclidean velocity,
unless the architecture is explicitly changed back to a weighted-loss-only setup.

No protected architecture conflict; this is the final optional VQ-surface-decoding experiment and leaves RouteCodePrior, MaskedCodeRefiner, MetricRefiner, `FlowNet`, `MetricNet`, and natural-velocity refinement unchanged.
- Added `SyntaxTokenRefiner` to `src/stage2_riemannian.py`, `scripts/train_syntax_token_refiner.py`, and `--syntax_refiner` support in `scripts/generate_deepseek_candidates.py`. This new optional branch keeps the rough VQ/RoutePrior draft generator frozen, feeds prompt latents plus rough decoded draft tokens/latents into a parallel token-space editor, and trains direct suffix-token CE with small draft-keep and anti-repeat losses so syntax repair is learned in surface-token space instead of code space. No protected architecture conflict; this leaves Stage1, VQ, RouteCodePrior, MetricRefiner, MaskedCodeRefiner, `FlowNet`, `MetricNet`, and natural-velocity refinement unchanged.
- Added invalid-token suppression for syntax-refiner training and generation: `scripts/train_syntax_token_refiner.py` and `scripts/generate_deepseek_candidates.py` now ban BERT special tokens and `[unused...]` tokens by default, with opt-out flags, and candidate generation can post-process adjacent repeated sampled tokens via `--avoid_adjacent_repeats`. No protected architecture conflict; this only constrains the token-space syntax-refiner output vocabulary and decoding cleanup.
